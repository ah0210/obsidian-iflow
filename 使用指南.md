## 快速开始

### 第一次使用

#### 步骤 1: 安装和配置

1. **安装 Obsidian**
   - 访问 [obsidian.md](https://obsidian.md/download)
   - 下载并安装适合你操作系统的版本
   - 打开 Obsidian，选择"打开文件夹作为库"
   - 选择 OrbitOS 的 `CN` 文件夹

2. **配置 iFlow**
   - 确保 iFlow CLI 已安装并可用
   - 在 OrbitOS 根目录下，确认 `iflow.config.json` 存在
   - （可选）根据你的需求修改配置

3. **熟悉界面**
   - 打开 Obsidian，查看左侧文件树
   - 熟悉各个文件夹的用途
   - 浏览 `99_系统/模板/` 中的模板文件

#### 步骤 2: 创建第一个每日笔记

```bash
iflow start-my-day
```

iFlow 会：
- 询问你今天的主要目标
- 询问是否有新想法或任务
- 询问是否有阻碍或顾虑
- 创建今日笔记 `10_日记/YYYY-MM-DD.md`
- 展示今日规划摘要

#### 步骤 3: 捕获第一个想法

在 `00_收件箱/` 中创建一个新文件：

```markdown
---
type: inbox
created: 2026-03-01
status: pending
tags: [inbox]
---

我想学习 TypeScript，掌握类型系统和高级特性
```

#### 步骤 4: 启动第一个项目

```bash
iflow kickoff 00_收件箱/学习TypeScript.md
```

iFlow 会：
- 读取收件箱项内容
- 询问项目详细信息
- 创建项目笔记 `20_项目/学习TypeScript.md`
- 归档原始收件箱项
- 链接到今日笔记

#### 步骤 5: 研究一个主题

```bash
iflow research TypeScript 高级类型系统
```

iFlow 会：
- 深度调研 TypeScript 高级类型
- 创建研究笔记 `30_研究/TypeScript高级类型系统.md`
- 提取核心概念到知识库
- 建立所有链接

### 日常使用流程

建议的每日工作流程：

1. **早晨** - 运行 `/start-my-day`
   - 回顾昨天进展
   - 规划今日任务
   - 处理收件箱

2. **工作期间** - 使用各种技能
   - 有问题？使用 `/ask`
   - 有想法？记录到收件箱
   - 需要研究？使用 `/research`
   - 想启动项目？使用 `/kickoff`

3. **项目执行** - 更新项目笔记
   - 完成任务，勾选清单
   - 添加进展更新
   - 链接到每日笔记

4. **每周** - 运行 `/archive`
   - 清理已完成的项目
   - 归档旧笔记
   - 整理收件箱

## 核心概念

### C.A.P. 项目结构

每个 OrbitOS 项目都遵循 C.A.P. 结构：

#### Context（背景）
- **目标**: 成功是什么样子
- **成功指标**: 可衡量的结果
- **限制条件**: 时间、资源、依赖

#### Actions（行动）
- **阶段**: 分解为多个阶段
- **任务**: 每个阶段的具体任务
- **优先级**: P0（紧急）到 P4（以后）

#### Progress（进展）
- **时间戳**: 每次更新的日期
- **链接**: 链接到每日笔记
- **状态**: 当前进展状态

### 双向链接

使用 `[[笔记名称]]` 创建链接：

```markdown
# 项目笔记

## 相关研究
- [[TypeScript高级类型系统]]

## 相关概念
- [[泛型]]
- [[条件类型]]
```

### Frontmatter 格式

所有笔记都使用 YAML frontmatter：

```yaml
---
title: "标题"
type: project|daily|inbox|wiki|research
created: YYYY-MM-DD
status: active|pending|processed|done|archived
tags: [标签1, 标签2]
area: "[[领域名称]]"
priority: P0|P1|P2|P3|P4
due: YYYY-MM-DD
---
```

## 最佳实践

   - 建立一天的工作节奏
   - 明确今日重点
   - 处理收件箱

2. **工作中** - 保持流动

   - 定期使用 `/ask` 快速查询
   - 完成任务后更新项目笔记

3. **晚上** - 简单回顾
   - 查看今日笔记

   - 为明天做准备

1. **周初** - 规划本周
   - 运行 `/start-my-day`
   - 查看所有活跃项目
   - 确定本周重点

2. **周末** - 归档整理
   - 运行 `/archive`
   - 清理收件箱

1. **启动项目**
   - 使用 `/kickoff` 创建项目

   - 标记所有任务为完成

   - 使用 `/archive` 归档项目

   - 建立概念之间的联系
   - 构建知识网络

## 常见问题

### Q: 我应该多久运行一次归档？

**A:** 建议每周或每月运行一次归档。归档会清理已完成的项目、旧的每日笔记和已处理的收件箱项，保持系统整洁。

### Q: 收件箱里的项应该什么时候处理？

**A:** 建议每天运行 `/start-my-day` 时处理收件箱。你可以选择：
- 使用 `/kickoff` 将想法转为项目
- 使用 `/parse-knowledge` 整理信息
- 使用 `/archive` 归档不需要的项

### Q: 如何处理长期未更新的项目？

**A:** 运行 `/archive` 时，iFlow 会识别长期未更新的项目（30 天以上）并询问你：
- 项目是否已完成？→ 归档
- 项目是否暂停？→ 更新状态为 `on-hold`
- 项目是否继续？→ 更新进展部分

### Q: 知识库笔记和研究笔记有什么区别？

**A:**
- **知识库笔记** (`40_知识库/`)：原子化的概念定义，可复用，简洁明了
- **研究笔记** (`30_研究/`)：深度分析，详细探讨，包含多个概念

研究笔记会提取核心概念创建知识库笔记，两者通过双向链接连接。

### Q: 如何在 Obsidian 中使用 iflow？

**A:** 有几种方式：
1. **在终端中运行**：在 OrbitOS 根目录下打开终端，运行 iflow 命令
2. **使用 Obsidian Terminal 插件**：安装 Terminal 插件，直接在 Obsidian 中运行命令
3. **使用快捷键**：为常用命令设置快捷键

### Q: 我可以自定义技能吗？

**A:** 可以！所有技能定义都在 `CN/99_系统/技能/` 目录下。你可以：
- 修改现有技能的工作流程
- 创建新的自定义技能
- 调整配置文件 `iflow.config.json`

### Q: 如何备份我的 OrbitOS 库？

**A:** OrbitOS 库只是一个文件夹，你可以：
- 使用 Git 进行版本控制
- 定期复制整个文件夹到备份位置
- 使用云同步服务（如 Dropbox、OneDrive）

### Q: 可以在多个设备上使用 OrbitOS 吗？

**A:** 可以！通过以下方式：
- 使用 Git 同步多个设备
- 使用云同步服务
- 使用 Obsidian Sync（付费服务）
